# Prompt to Canvas

Agent skill for turning notes, project stories, plans, metrics, and technical
concepts into polished, editable Excalidraw canvases.

Prompt to Canvas is a design-system skill, not a template filler. The agent
reads the user's content, chooses a fitting visual style from a 35-style
catalogue, composes an SVG layout, converts it into editable Excalidraw scene
JSON, and opens it in a bundled local editor with PNG/SVG export.

## Features

- **Editable Excalidraw output** - generated boards remain editable shapes and
  text, not screenshots.
- **35 curated visual styles** - restrained, balanced, and bold palettes with
  style previews and design guidance.
- **Local editor runtime** - the skill bundles one clean static editor build, so
  users do not need a hosted service.
- **Bilingual editor UI** - Chinese and English UI switching.
- **Refresh-safe autosave** - local browser autosave for the current scene.
- **PNG/SVG export** - direct export from the bundled editor.

## Install

Clone this repository and copy the skill directory into your agent's skill
folder.

For Codex:

```bash
mkdir -p ~/.codex/skills
cp -R src/skills/prompt-to-canvas ~/.codex/skills/
```

For Claude Code:

```bash
mkdir -p ~/.claude/skills
cp -R src/skills/prompt-to-canvas ~/.claude/skills/
```

You can also use the deployment helper. It defaults to `~/.codex/skills` and
can be overridden with `SKILLS_DIR`.

```bash
scripts/deploy.sh
SKILLS_DIR="$HOME/.claude/skills" scripts/deploy.sh
```

## Use

Ask your agent in plain language:

```text
Use prompt-to-canvas to turn these project notes into a polished editable canvas.
```

Or describe the output:

```text
把这段项目经历做成一张适合面试 portfolio 的可编辑可视化图。
```

The skill will pick a style, build an SVG source, convert it to Excalidraw scene
JSON, validate the scene, and open a local editor URL.

## Development

Install dependencies:

```bash
npm ci
```

Run the editor in development mode:

```bash
npm run dev
```

Build and sync the bundled skill editor runtime:

```bash
npm run build:skill
```

Run the skill preflight check:

```bash
npm run preflight
```

## Repository Layout

```text
.
├── docs/                         # Product, technical, user, and release docs
├── scripts/                      # Deployment helper scripts
├── src/
│   ├── editor/                   # Vite + React editor source
│   └── skills/
│       └── prompt-to-canvas/     # Distributable skill package
│           ├── SKILL.md
│           ├── CATALOG.md
│           ├── RULES.md
│           ├── assets/
│           │   ├── editor/       # One clean prebuilt editor runtime
│           │   └── styles/       # 35 PNG style previews
│           ├── rules/
│           ├── scripts/
│           └── templates/
├── LICENSE
├── NOTICE.md
├── package.json
└── package-lock.json
```

`src/static/` is generated by Vite and intentionally ignored by git. The
publishable runtime copy lives in
`src/skills/prompt-to-canvas/assets/editor/`.

## Release and Size Policy

See [docs/RELEASE.md](docs/RELEASE.md).

The expected skill package size is about 14 MB:

- About 8 MB for the bundled Excalidraw editor runtime.
- About 5 MB for the 35 style preview PNG files.
- Less than 1 MB for skill rules, scripts, templates, and documentation.

Do not publish `node_modules/` or `src/static/`.
